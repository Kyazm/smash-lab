# リサーチ結果と設計への反映

既存ツール・AI試合分析・esports学習科学・プロの実践に関するdeep research（5系統・ソース23件・claim 115抽出→25件を3票敵対検証→確定11件）の要約と、設計への反映。実施: 2026-07-03。

## 発見 → 設計への反映

| # | 発見（検証済み） | 設計への反映 |
|---|---|---|
| 1 | **outofshield.com** が確定反撃検索の完全な先行事例。①マッチアップ検索 ②キャラのOoS一覧 ③逆引き「自分のどの技がガードに安全か」の3モード。「フレーム上確定でも間合いで空振りする」と運営自身が注記 | F3を**双方向化**: 守り(相手技→自分の確反)に加え、攻め(自技が相手ガードに安全か)を追加。リーチ注記の方針も裏付け（ADR-0006） |
| 2 | **ultimateframedata.com** に確反計算の全数値が揃う。キャラ別OoS発生一覧（Misc Info/statsページ）まで公開。投げ/回避にシールド系フィールドがないのは仕様上正しい省略 | oos_optionsを**全キャラ分**保持し、UFDから初期取込。ZSS分のみ手動で精緻化 |
| 3 | **Rubendal SSBU-Calculator**（OSS）が精密計算のパラメータ面を定義: 1v1の1.2倍補正・DI/LSI・ワンパターン相殺等。同梱データは「非OP相殺・単発・持続初F当て」前提 | 計算前提の注記をUIに明示（反映済み）。キル%計算を将来拡張する場合はこのOSSから数式を移植 |
| 4 | 日本の**検証窓スプレッドシート**は崖行動・受け身・起き上がり・転倒のフレームまでカバー。ただし全キャラ網羅のclaimは反証(1-2)されたためカバレッジ要実地確認 | 日本語技名マッピングの参照元に採用。崖攻防・受け身読みの確定行動判定は将来拡張として記録 |
| 5 | **Slippipedia**（Melee/Slippi）が目標アーキテクチャを実証: ローカル解析+永続DB+punish単位インデックス+**統計→該当場面ジャンプ→MP4クリップ書き出し**。ただしUltimateには構造化リプレイデータがなく動画側から復元するしかない（最大の技術ギャップ） | AIレビューのfindingsから**該当場面ジャンプ+ffmpegクリップ書き出し→メモ添付**を追加。CV/OCRによるpunish単位インデックスは未解決課題として保留 |
| 6 | 既存キャラ対メモアプリ（Smash Notes、5,000+DL）のユーザー要望: キャラ/マッチアップに加え**特定プレイヤー粒度**のノートが必要。リッチメディアと階層/タグ整理が最多要望。アプリは2023年から未更新でカテゴリ未成熟 | notesに**playerスコープ**を追加（同キャラでもプレイヤーごとに動きが違う）。メディア/タグは設計済みで裏付け |
| 7 | **SSBU-Matchups**: マッチアップは方向別ドキュメント+再利用セクション分類（ニュートラル/不利状況/復帰阻止/飛び道具対策/ステージ選択）。設計根拠は「**対戦の合間に数秒でスキャンできること**」 | キャラ対メモに**セクションテンプレート**と**要点ピン留め(TL;DR)**を追加。スマホUIはスキャン速度優先 |
| 8 | Gemini動画理解は**デフォルト1fpsサンプリング**で高速アクションの詳細を見逃すと公式明記。fpsパラメータで引き上げ可。約300トークン/秒（保守的上限） | パイプラインは既定1fps=立ち回りレビュー用と明記。対話深掘りで**区間指定の高fps再解析**を追加 |
| 9 | **GameplayQA**ベンチ: 最良MLLM 71.3% vs 人間80.5%。ゲーム動画理解でAIは人間に有意に劣る | findingsに**承認/棄却**操作を追加。「AIの指摘は候補」というUI前提。癖統計は棄却分を除外 |
| 10 | 格ゲー実証研究(SFV対空、n=3): 「1スキルに絞った動画フィードバック→実戦転移」が改善を実証（例: 対空実行率0-5%→54%→ランク戦66%） | focus_pointsは**アクティブ1〜3個に制限**。達成度をfocus_evaluationsから**推移グラフ**で追跡 |
| 11 | LoLプロ定性研究(N=10): VODレビューは「自分とトップの試合の**差分比較**」として使われる。「完全集中の3試合は無目的な15試合より10倍良い」 | intelで収集したトップZSSの試合動画をマッチアップ別に「**参考VOD**」として紐付け。**セッション**概念（目的設定→振り返り）を追加 |

## 未解決課題（実装時に検証）

1. Gemini fps引き上げ時（10-30fps）のスマブラ動画の状況ラベリング精度 — 公式推奨はあるが格ゲー実測データなし。Phase 3冒頭で実測
2. 動画からのCV/OCRパイプライン（%表示・ストック・キャラ認識）によるpunish単位インデックス — Slippipedia相当をUltimateで実現する唯一の道。既存実装の有無は未調査
3. 技のリーチ数値データの所在 — 「届く確反」への絞り込みに必要。UFDはGIFのみ
4. スマメイトの戦績データへのプログラムアクセス可否 — playerスコープノートとの紐付け

## 注意事項

- 学習科学のエビデンスは小サンプル（n=3、N=10）でスマブラSP直接の研究は皆無。設計方針の根拠にはなるが効果量の保証ではない
- Geminiのトークン/サンプリング仕様は変更されうる。300トークン/秒で予算化（Gemini 3系は実効約100/秒）
- 参照サイト（UFD、outofshield.com等）は保守停止リスクあり。データは必ずローカルに取り込む（設計済み）
- スマメイト・insights.gg等の一次調査は3票検証を通過せず、本レポートは主張しない
