# smash-lab 要件定義

スマブラSP上達のための個人用ナレッジ&分析システム。
現在Discordサーバーで運用しているキャラ対メモ・技考察を置き換え、フレームデータ参照・AI試合レビュー・情報収集を統合する。

**自キャラ（メインキャラ）はアプリ内で選択可能**（ADR-0013で後から追加。初期値・以下の例はゼロスーツサムス＝ZSS）。自キャラを切り替えると立ち回り/技/試合メモは各キャラ単位で保持される（非破壊）。本書中の「ZSS」表記は現在の自キャラの例であり、機能はメインキャラ非依存。

## 利用者・利用シーン

- 利用者は本人のみ（シングルユーザー）
- 対戦中・対戦の合間にスマホ/iPadでキャラ対メモを確認する
- 対戦後にMacで録画をレビューし、反省点を蓄積する
- 新しいキャラと当たる前にPC/スマホで対策を予習する

## 機能要件

### F1. 自キャラページ（メインキャラは選択可、初期値ZSS）
- 立ち回りメモ・技ごとの考察メモを作成/編集できる（Markdown）
- 画像・動画クリップを埋め込みできる
- 現Discordの「ゼロサムについて」「ゼロサム技」カテゴリの置き換え

### F2. 全キャラフレームデータ
- 全ファイターの技ごとに 発生F / 持続F / 全体F / ガード硬直差 / ダメージ を閲覧できる
- ヒットボックス画像を表示できる
- 技名は日本語表記（英語名も併記）
- データ取得元: ultimateframedata.com（Ver.13.0.1 最終。取込は一度きり、更新同期は不要）

### F3. 確定反撃判定（双方向）
- **守り**: 相手キャラの技を選ぶと、その技をガードした後にZSSのどの行動が確定するかを一覧表示する（実効発生Fと猶予F付き）
- **攻め**: ZSSの技を選ぶと、相手キャラのOoS性能と比較して「ガードされても安全か/何で反撃されるか」を表示する
- 前提条件（密着ガード・持続の先端当てで変動する等）を注記として明示し、「フレーム上確定（リーチ・位置要確認）」の表現とする

### F4. キャラ対メモ
- 相手キャラごとにメモを作成/編集できる（Markdown、画像/動画埋め込み可）
- 現Discordの「64キャラ対策」「DXキャラ対策」等カテゴリの置き換え
- 重要度マーク（現Discordの⭐相当）を付けられる
- **セクションテンプレート**（ニュートラル/不利状況/復帰阻止/飛び道具対策/ステージ選択）を任意適用できる
- **要点ピン留め（TL;DR）**: 対戦の合間に数秒でスキャンできるよう、キャラ対ページ冒頭に要点を固定表示できる
- **プレイヤー粒度のメモ**: 同じキャラでもプレイヤーごとに動きが違うため、特定プレイヤー宛のメモを作成できる
- 既存Discordデータは全量自動移行する（F8）

### F5. キャラ名検索
- キャラ名（日本語/英語）で検索し、そのキャラのフレームデータ・キャラ対メモ・外部収集情報に到達できる
- 技名・タグ・メモ本文・外部収集情報の表題/要約も検索対象とする（結果表示は自分メモと外部情報でレーンを分ける）

### F6. AI試合レビュー
- キャプチャボードからYouTubeに限定公開でアップロードされたフル試合動画を入力とする
- 動画URLを登録すると自動で解析され、レビューレポートがDBに蓄積される（自チャンネルの新規アップロード自動検知は拡張）
- レポート内容: タイムスタンプ付き指摘、状況分類（ニュートラル/有利/不利/崖/復帰阻止）、癖の検出
- 「意識すること」リスト（focus points）を登録でき、AIは各試合でその達成度を評価する。アクティブな意識点は**1〜3個に制限**し（学習科学の裏付け: 1スキル集中→実戦転移）、達成度の推移をグラフで追跡できる
- AIの指摘（findings）は候補扱いとし、**承認/棄却**できる。癖の統計は棄却分を除いて集計する（MLLMの精度は人間未満という前提）
- 指摘はYouTubeのタイムスタンプにリンクし、**該当場面へ直接ジャンプ**できる。メモへはURL+開始秒で場面を埋め込める
- 蓄積レポートを横断した癖の統計を閲覧できる
- 気になる試合はClaude Codeで対話深掘りできる（区間指定の高fps再解析を含む）
- 相手キャラ・勝敗のAI推定は初期値であり、レビュー画面から手動訂正できる（癖統計は訂正後の値を使う）
- 解析エンジンはGemini API（動画ネイティブ入力）。精度の期待値: フレーム単位の技術指摘は保証しない。癖・傾向・状況分類の検出を主目的とする

### F7. 外部情報収集（ZSS特化）とメモ更新
- 週次の定期タスクがZSS関連の対策記事・動画をクローリング・要約し、フィードに蓄積する
- キャラ対ページから「このキャラの対策を深掘り」をオンデマンド実行できる（非同期。ローカル機での処理完了後にフィードへ反映）
- **収集情報は自分のメモと分離する**: 出典付きの受信箱（inbox）レーンに入り、自分が「採用」した時のみメモへ反映される。破棄（dismissed）も選べる
- 採用時は2通り: (a) 新規ノート化、(b) **既存メモへの追記案** — AIが追記先と文面を提案し、差分プレビューを承認した時のみ既存メモが更新される（出典付き）。承認なしに既存メモが書き換わることはない

### F8. Discordデータ移行（一度きり）
- DiscordChatExporterで全チャンネルをJSONエクスポートし、新スキーマへ構造化投入する
- チャンネル名からキャラ/技カテゴリを自動判定する
- 添付画像・動画も回収し、メディアストレージへ移す

### F9. 練習セッション管理
- セッション（日付+目的）を作成し、試合を紐付けられる
- セッション前に目的を設定し、後に振り返りを書く軽量フロー（プロの実践知見: 完全集中の少数試合 > 無目的な多数試合）
- 目的設定は**プロセス目標**を促す（結果目標入力の言い換えナッジ）。振り返りに必須質問、連敗時のティルト検知→休憩提案、分散練習の可視化を含む（詳細仕様: docs/05_practice-science.md）
- focus_pointsは技術/メンタルの2カテゴリ。AIレビューの指摘からトレモドリル案への導線を持つ
- 収集した**トッププレイヤーの参考VOD**をマッチアップ別にキャラ対ページへ紐付け、自分の試合と見比べられる（差分比較レビュー）

## 非機能要件

- **コスト**: ランニングは無料枠+Gemini API従量（目安: 1試合数円〜十数円）に収める
- **デバイス**: 閲覧系はスマホ/PCブラウザ対応（レスポンシブ）。AI処理系はMacローカルのみで可
- **認証**: ホスト公開するため単一ユーザー認証必須。本人以外はデータへアクセス不可
- **データ所有**: メモ・レビュー結果はエクスポート可能な形式（Postgres/Markdown）で保持する
- **保守**: フレームデータは静的。定期実行が止まっても閲覧系は影響を受けない疎結合とする

## スコープ外

- スマブラSP（Ultimate）以外のシリーズのデータ
- 対戦の自動記録（Switchからの自動取込はキャプチャボード録画で代替）
- リプレイファイル解析（スマブラSPにはSlippi相当の仕組みがない）
- 他ユーザーへの公開・共有機能
