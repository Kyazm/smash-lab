# ADR-0016: ポケトレ/ホムヒカを1枠に畳むキャラグループ

Status: Accepted (2026-07-04)

## Context
スマブラSPのキャラ選択画面では、ポケモントレーナー(ゼニガメ/フシギソウ/リザードン)とホムラ/ヒカリは1枠で、選択後に交代する。一方フレームデータの取込元(UFD)は各ファイター個別で、`characters` テーブルにも `pt_squirtle`/`pt_ivysaur`/`pt_charizard`/`pyra`/`mythra` の5エントリが別々に入っている。一覧が89体でバラけており「キャラ選択画面と同じにしたい」という要望が出た。

## Decision
データ(characters/moves/oos_options)は個別ファイターのまま変えず、**フロント側の表示と集約だけを畳み込む**。グループ定義は `web/src/lib/characterGroups.ts` に純データとして持つ（ポケトレ=代表 pt_squirtle、ホムヒカ=代表 pyra）。

- **一覧**: グループは代表位置に1枠だけ表示（89→86枠）。表示名はグループ名、勝敗記録は代表idに集約。検索は全メンバー名も対象。
- **キャラ詳細**: グループメンバーのページに「ゼニガメ/フシギソウ/リザードン」サブ切替を出し、選択で該当slugへ遷移（tab維持）。フレーム表・確定反撃は各ファイター個別（別々の技のため）。
- **戦績・キャラ対メモ**: 「1キャラ（対戦相手）」として代表idに集約する。試合中に交代するため対戦相手としては1つ、という実態に合わせる。サブキャラを切り替えても戦績・メモは共通。
- **/stats ランキング**: 対戦相手idを `makeGroupResolver().normalizeId()` で代表idに正規化して集計し、表示名をグループ名にする。

## Consequences
- フレームデータのスキーマ・取込パイプラインは無変更。畳み込みはUI層に閉じる。
- 戦績の集約先は「代表ファイターid」。一覧・詳細・ランキングすべてで代表idに正規化するため、個別ファイターidに戦績が溜まることはない（既存データも正規化時に代表へ寄る）。
- 自キャラ(is_main)判定は代表ファイター基準。ポケトレを自キャラにする場合は代表(ゼニガメ)で設定する想定。
- 将来キャラ選択画面の並び順そのものを再現したくなった場合は、このグループ定義に順序メタを足して拡張できる。
