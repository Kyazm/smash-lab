# ADR-0004: 自分メモと外部収集情報の分離（inbox昇格モデル）

Status: Accepted (2026-07-03)

## Context
外部から自動収集した記事/動画要約が自分のキャラ対メモに混ざると、自分の検証済み考察が汚染される（ユーザー明示要望）。

## Decision
- `notes`（自分の考察）と `intel_items`（外部収集）をテーブル・UIタブともに分離
- 収集情報は `status=inbox` で溜まり、ユーザーの「採用」操作でのみメモへ反映。破棄も可
- 採用は2モード: `new_note`（新規ノート化）/ `append_proposal`（既存メモへの追記案。差分プレビューをユーザーが承認した時のみ反映）。いずれも `source=intel_adopted`+出典URL付き
- 昇格経路はDB関数 `adopt_intel()` に一本化。パイプラインコードは `notes` への書込を実装しない
  （service roleはRLSをバイパスするため、この禁止はDB権限では担保されない。パイプラインのコードレビュー項目として運用で担保する）

## Consequences
- メモの信頼性が保たれ、出典を遡れる
- 採用操作という一手間が増える（意図的なコスト）
- 「クローリングでメモをアップデート」要望は追記案の承認制で満たす（無断書換なし）
- ユーザーのmemory-dream運用（受信箱→昇格）と同型で認知負荷が低い
