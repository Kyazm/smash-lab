# ADR-0017: VIPランク計算（世界戦闘力→段位）と外部データの扱い

Status: Accepted (2026-07-04)

## Context
VIPモードの世界戦闘力から、クマメイト(https://kumamate.net/vip/)のように段位を出したいという要望。クマメイトは「段位しきい値 = 推定VIPボーダー × 段位指数（VIP到達=1.0基準）」で、絶対値でなくボーダー基準の相対倍率でランクを決めている。

課題は2つ: (1) このアプリはGitHub Pages（静的）で、ブラウザから kumamate.net を直接fetchするとCORSで不可（実測 "Failed to fetch"）。(2) 外部データの非転載方針（ADR-0011）との整合。

## Decision
- **正規化ロジックと段位テーブルはローカルに持つ**（`web/src/lib/vipRank.ts`、純関数＋テスト）。段位名・段位指数（28段階）はクマメイト準拠で、UIに**出典を明記**する。段位指数は「計算式のパラメータ」であり、ADR-0011が避けたい「大量の実データ転載（スマアナのMU統計等）」とは性質が異なると判断。
- **VIPボーダー実測値は自動取得しない（手入力＋プリセット）**。ボーダーはクマメイト管理人の推定・更新の労力の核心であり、これを自動scrape・再配布するのは方針・礼儀・CORSの全てに反する。UIは初期値プリセット（端末保存）＋「クマメイトを開く」リンクで最新確認を促す。ボーダーは1分3程度しか動かず段位境界は数%刻みのため、手入力値が多少古くても段位はほぼ変わらず実用十分。
- 配置は `/stats` の「全体/VIP」フィルタ時にセクション表示（戦績0でも使える独立計算）。

## Consequences
- 外部サーバーへのリクエストは発生しない（CORS・負荷・規約リスクを回避）。
- 段位はクマメイトの推定を推定で掛ける方式のため参考値（クマメイト自身も同旨を明記）。段位指数テーブルは2020年更新ベースで、分布が変われば乖離しうる。
- 将来ボーダーを自動更新したくなった場合の選択肢: Supabase Edge Functionプロキシ、またはMacローカルのAI処理系で定期取得しSupabaseに1行保存（低頻度・キャッシュ・出典明記が前提）。本ADRのスコープ外。
- キャラごとのVIPボーダー差は段位計算に反映しない（全体ボーダー基準。クマメイトの段位も同様）。
