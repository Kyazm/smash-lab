# ADR-0007: 試合動画はYouTube限定公開を正とし、レビューはタイムスタンプ参照

Status: Accepted (2026-07-03) — ADR-0005の「フル試合録画=Macローカル」を置き換え

## Context
実運用ではキャプチャボードの録画はYouTubeへ限定公開で自動アップロードされる。Gemini APIはYouTube URLの直接入力に対応するが**公開動画のみ**で、限定公開は受け取れない。

## Decision
- 試合動画の正はYouTube限定公開。Macローカル保管はレビュー時の一時キャッシュのみ
- レビューパイプラインはyt-dlpで一時ダウンロード（自分のコンテンツ）→Gemini File API→解析後に一時ファイルとFile API上のファイルを削除
- findingsの `t_sec` はYouTubeタイムスタンプリンク（`watch?v=…&t=Ns`）と埋込プレイヤーのシークで参照。ffmpegクリップ切り出しは不要（メモへはURL+開始秒で埋込）
- 動画の検知はWeb UIでのURL登録を基本とし、YouTube Data API(OAuth)による自チャンネル新規アップロードの自動検知を拡張とする

## Consequences
- フル動画のローカルパス管理・散逸リスクが消え、スマホからも試合動画そのものを見られる
- クリップ切り出しパイプラインが不要になる（タイムスタンプ埋込で代替）
- yt-dlpはYouTube仕様変更で壊れうる（保守リスク）。壊れた場合は手動ダウンロード→ローカルファイル指定のフォールバック
