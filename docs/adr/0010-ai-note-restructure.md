# ADR-0010: キャラ対メモのAI整頓（提案・承認制）

Status: Accepted (2026-07-03)

## Context
Discordから移行した63件のキャラ対メモは時系列の散文で、対戦の合間の数秒スキャンに向かない（ユーザー指摘）。一方、ADR-0004の通り自分の考察はAIに無断で書き換えさせない方針。

## Decision
- **提案・承認制**: パイプラインが対象メモ（`kind='matchup'` 63件。own系は構造が保たれているため対象外）の整頓案を `note_proposals` に生成し、ユーザーがアプリ内の差分ビューで1件ずつ承認/却下。一括承認・部分承認は作らない（提案は原子的）
- 整頓ルール: ①事実の追加・削除・意味変更を禁止（再配置と重複統合のみ）②TL;DR+セクション構成へ ③日付は出典注記に縮約 ④矛盾・古い可能性は削除せず⚠マークで並記
- **楽観ロック**: 提案は生成時点の `notes.updated_at` を `base_updated_at` として保持。`apply_note_proposal` は適用前に照合し、不一致なら `status=stale` で中断（承認前の手動編集を上書きしない）。stale提案はUIから個別再生成
- 承認時は `note_revisions` に旧本文を保全してから更新（RPCで1トランザクション）。migration 0002。note_revisionsは将来のintel採用（ADR-0004 append_proposal）でも共用できる汎用設計とする

## Consequences
- メモがスキャン可能になり、原本はいつでも復元可能
- 63件の承認作業が発生する（意図的なコスト。自分の考察の変質を防ぐ最後の砦）
- 同じ仕組みを将来 intel の append_proposal（ADR-0004）にも転用できる
