# 練習科学リサーチ — エビデンスのある練習方法と心構え

スマブラの練習法にまつわる通説を、心理学・脳科学の研究25件で検証し、根拠のある方法だけをまとめる。

**この記事でわかること**

- 分散練習・下位スキル分解など、同じ練習時間でも差がつく練習の組み方
- 「勝つ」ではなく行動の目標を立てる（プロセス目標）ことの効果の大きさ
- ティルト（怒りや焦りで崩れる状態）への現実的な対処法
- プレッシャー下でミスを減らす注意の向け方
- 「量をこなすだけでは無意味」など、研究で否定された通説

deep research（5系統・25ソース・121claim→25件を3票敵対検証→確定9件・反証5件）。実施: 2026-07-03。
エビデンス強度ラベル: 【実証研究】=査読付き研究が直接支持 /【相関】=観察データのみ /【通説】=コミュニティ実践（本調査では未検証）。

## 練習原則

1. **分散練習**【実証研究】: 同量の練習をするなら、集中して一気にやるより分散させる（短時間×高頻度）方が保持・成績に優れる。85万人規模のゲームデータでも確認された。効果量（研究で示された変化の大きさを表す数値。目安としてd=0.2は小さい、0.5は中くらい、0.8は大きい、と読む）は小さいd≈0.11だが、方向は頑健である
2. **下位スキルへの分解+段階的難易度+即時フィードバック**【実証研究】: 専門家は試合全体を漫然と反復するのではなく、困難な部分だけを切り出して集中的に反復する。この考え方は、トレモでの局面切り出しドリルを直接正当化するものだ。注意: 外的フィードバックが過剰に頻繁だと、かえって保持を損ないうる
3. **時間カウントより質と目的**【実証研究】: CS:GOを52週間追跡した登録報告研究（登録報告＝分析前に仮説と手法を公開し、後付けの結果解釈を防ぐ研究形式。N=597）では、deliberate practice（漫然と量をこなすのではなく、弱点を意識して行う意図的な練習）の「時間」は成績と無相関だった（r=.02/-.01）。ただし**「量が無意味」も反証済み**で、LoLでは通常の対戦経験量がr=.26で成績を予測した。実戦も練習のうちと言える
4. **初期は実験が有利（可能性）**【相関】: 上達初期に多様な戦略を試すプレイヤーほど後の成績が高い傾向がある。ただし後続研究では因果関係は限定的とされており、参考程度に留めておきたい

## 心構え・メンタル

5. **プロセス目標**【実証研究・最重要】: プロセス目標とは、「勝つ」のような結果ではなく、「崖上がりの読みを散らす」「OoS（ガード中からの反撃行動）の選択肢を増やす」のように、自分の行動そのものを目標にすることを指す。目標設定メタ分析でプロセス目標d=1.36、結果目標d=0.09だった（15倍差）。**目標設定→自己モニタリング→振り返りの自己調整サイクル（目標を立てる→実行しながら自分を観察する→終わったら振り返る、というサイクルを自分で回すこと）はd=1.53**で最大の効果。smash-labの中核ループ（セッション目的設定→振り返り+focus_points）はこれと直接整合
6. **ティルト（怒りや焦りで平常心を失い、判断が乱れる状態）対策は「距離置き」**【相関、N=1,007】: ティルトの最大予測因子は怒り。「勝つため」動機は「楽しむため」よりティルトが強い。測定された感情制御方略のうち**休憩して別の活動をする（distraction）だけが有効**だった
7. **ティルト耐性は変えられるという信念**【相関】: growth mindset（ティルトしやすさは訓練で変わるという考え方）を持つ人ほどポジティブな対処を選ぶ。最も強いネガティビティは自分自身に向く
8. **感情制御は第3の専門技能**【質的研究】: メカニクス・メタゲーム理解と並ぶ学習可能なスキルであり、固定特性ではない。技術系と同格に練習対象として扱うべき
9. **プレッシャー下は外的フォーカス**【相関2-1票】: 手元の操作を意識的にモニタリングする傾向（Reinvestment、自分の動作を意識しすぎること）はchoke（プレッシャー下でのミス・崩れ）傾向と正相関。入力の指ではなく相手の行動・状況に注意を置くとよい——たとえば自分の指の動きでなく、相手の着地際の暴れ択に注意を向ける、など

## 反証された通説（アプリに持ち込まない）

- ×「量のグラインドは無意味、deliberate practiceだけが上達させる」（1-2で反証。実戦量も効く）
- ×「ただ遊ぶだけでは上達しない」（0-3で反証）
- ×「経験を積めばプレッシャー下でclutch（土壇場で結果を出すこと）できるようになる」（1-2で反証）

## smash-labへの組み込み仕様

| 機能 | 仕様 | 根拠 |
|---|---|---|
| セッション目的設定 | 結果目標らしき入力（「VIP到達」「勝つ」）を検知したらプロセス目標への言い換えを促す | #5 |
| 振り返り必須項目 | 「設定したプロセスを何回実行できたか」+「自分を過度に責めていないか」 | #5, #7 |
| focus_pointsカテゴリ | `technical` と `mental` を同格で用意。mental例:「ストック先行されても投げやりバーストを狙わない」「リード時に攻め急がない」 | #8 |
| focus_pointの粒度 | 「崖狩りの2択」「ガード後の確反」のような下位スキル単位を促すプリセット | #2 |
| AIレビュー→ドリル導線 | 問題局面の特定→「トレモで再現するドリル案」を提示 | #2 |
| ティルト検知→休憩提案 | 連敗・振り返りでの感情悪化を検知したら「一旦離れて別のことを」を提案。ティルト管理は訓練可能なスキルというUIコピー | #6, #7 |
| 分散練習の可視化 | 前回セッションからの間隔と頻度を表示。「今日は切り上げて明日」を肯定するコピー。時間量のゲーミフィケーション（時間バッジ等）は**採用しない** | #1, #3 |
| 実戦/トレモ比率 | セッションに実戦・トレモの内訳を記録。実戦も練習として肯定 | #3 |
| 試合前ルーティン | 「相手の行動選択に注意を置く」外的フォーカスのリマインダーをプリセットに | #9 |
| 実験セッション | 「普段使わない択を3回試す」テンプレート（初中級向け、効果は限定的と注記） | #4 |

## 注意事項・未カバー領域

- esports系知見の大半は相関・横断・自己報告。因果を示す介入研究はほぼなく、スマブラ/格ゲー固有の実証研究は確定claimに1件もない（SFV対空研究はdocs/04 #10参照）
- ティルト研究はチームMOBA由来。1v1での誘因構造（自分のミス vs 相手の戦法）は直接データなし
- **未検証のまま残った領域**: Sirlin『Playing to Win』、メンタルスタック理論、対戦ノート運用、日本語コミュニティのコーチング実践、睡眠と運動学習、ビジュアライゼーション。これらは【通説】ラベルで扱い、断定しない。日本語ソースは確定claimに残らなかった
