# smash-lab 実装計画

フェーズごとに単体で価値が出る順に積む。各フェーズ完了時に動くものがある状態を維持する。

> **実装状況（2026-07-03）**: Phase 1・2 は完了・本番稼働。加えて計画外の追加機能を多数実装済み（自キャラ選択=ADR-0013、ゲストサンドボックス=ADR-0014、戦績記録〈モード別・マッチアップ別の勝敗ログ+勝率/連勝グラフ〉=ADR-0015、練習ループ〈意識ポイント/セッション/ティルト検知〉=ADR-0018、公開ライブラリ/記事、確反のジャスガ・ガーキャン空中技分離・ガード解除具体技化、メモのリッチ埋め込み、デザイン刷新〈黒×赤白×黄〉、キャラアイコン外部参照、CI/CD整備）。Phase 3（AI試合レビュー）・Phase 4（外部情報収集）は未着手（Gemini課金待ち）。詳細な設計判断は docs/06〜08 と ADR を参照。

## Phase 1: データ基盤 + 閲覧コア（フレームデータ/確定反撃/検索）

- Supabaseプロジェクト作成、スキーマ・RLS・Auth（サインアップ無効化、メールOTP/MFA）
- **着手条件（解消済み 2026-07-03）**: 取込元調査完了。GitHub既存データセット（rubendal等）はガード硬直差の完成値を持たないため、数値フレーム/硬直差/hitbox画像=UFDスクレイプ、日本語技名=検証窓シート（gviz CSVでプログラム取得可、実質フルロスター確認済み）、OoS一覧=UFD statsページで確定。英日技名の突合は技の出現順ベース+人力レビュー
- `import-framedata`: 確定した取込元 → characters/moves投入
  - 日本語技名は検証窓スプレッドシート（信頼ソース）から取得し、不足分のみAI生成 → `.context/` のCSVを人間確認 → 投入
  - ヒットボックス画像は当面UFDへのリンク表示。ミラーする場合は認証必須の非公開バケット限定（ADR-0003改訂: 公開再配布しない）
  - 全キャラの `oos_options` をUFDのOoS一覧から自動取込（ADR-0006）。ZSS含め自動を正とし、上B/掴み等の代表値のスポットチェックのみ実施
- Webアプリ骨格: キャラ一覧/検索 → キャラページ（フレーム表 + 確定反撃タブ〈守り/攻め〉）
- GitHub Pagesデプロイ

完了条件: スマホから「相手の技→確定反撃リスト」と「自分の技→ガードに安全か」が引ける。

## Phase 2: メモ機能 + Discord移行

- notes/note_media CRUD（Markdownエディタ、画像アップロード、YouTube埋込）
- 自キャラページ（メインキャラは選択可、初期値ZSS）: 立ち回りメモ/技別メモ
- キャラ対ページ: メモタブ（⭐、タグ、本文検索、セクションテンプレート、TL;DRピン留め）
- プレイヤー粒度メモ（kind=player）
- `import-discord`: エクスポート→マッピング確認→投入
- 横断検索（キャラ名/技名/タグ/本文/intel）

完了条件: Discord運用を停止できる。スマホでキャラ対ページのTL;DRが数秒でスキャンできる。

## Phase 3: AI試合レビュー

- `review-match` パイプライン（URL登録→yt-dlp取得→ffmpeg→Gemini→後始末→書込）
- 解析プロンプト設計: 構造化出力(findings/summary/focus_evaluations)、habit_tags固定語彙
- **冒頭で精度実測**: 1fpsでの状況ラベリング精度と、区間高fps再解析（fps=5〜10）の効果を10試合で検証し、指摘粒度を調整（docs/04 未解決課題#1）
- focus_points管理UI（アクティブ1〜3個制限、達成度推移グラフ）
- レビュー閲覧UI: opponent/result手動訂正、findings承認/棄却、該当場面ジャンプ
- 場面参照: findings→YouTubeタイムスタンプリンク/埋込プレイヤーのシーク。メモへのURL+開始秒埋込
- 対話深掘りモード（Claude Code、区間指定高fps再解析）
- セッション管理（目的設定→試合紐付け→振り返り）。セッション/focus_point UIは docs/05 の組込仕様表に従う（プロセス目標ナッジ、振り返り必須質問、ティルト休憩提案、分散練習可視化、時間ゲーミフィケーション不採用）

完了条件: 録画を置くだけでレポートが溜まり、承認済みfindingsベースの癖統計が見える。

- 参考VOD欄（トッププレイヤー動画のマッチアップ別紐付け、自分の試合との比較ビュー）

完了条件: 週次でZSS情報が受信箱に届き、採用フローが回る。

## リスクと対策

| リスク | 対策 |
|---|---|
| UFDのHTML構造が想定と違いスクレイプ難航 | 先にGitHub既存JSONを調査、あれば取込元を差し替え(ADR-0003の代替案) |
| Gemini動画解析の精度不足 | 期待値を癖検出に限定済み。focus_points評価とタイムスタンプ精度をPhase 3冒頭で実測し、だめなら指摘粒度を下げる |
| yt-dlpがYouTube仕様変更で壊れる | 定期更新。壊れた場合は手動ダウンロード→ローカルファイル指定のフォールバック |
| GitHub Pages(公開リポ)での運用 | リポジトリにはコードのみ。データ・キーは一切コミットしない。プライベートリポ+Actions Pagesも可 |
