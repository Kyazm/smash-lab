# 試合の振り返り方

全部見返す必要はない。撃墜シーンだけを深掘りして、直す点を1つに絞るのが一番効く。

## この記事でわかること

- 振り返りで実際に見るべき場面（全部ではない）
- 実行ミスと判断ミスの見分け方
- 1回の振り返りで直していいのは1つだけ、というルール
- smash-labのAIレビュー機能の使い方

## なぜ振り返るか

試合中の判断のほとんどは、頭で考えて出しているわけじゃない。反射でやっている「手癖」だ。振り返りの目的は、その手癖に後から理由を付け直すこと。「なんとなく」で通した行動を洗い出して、次から意図的な選択に変えていく作業だと考えるといい。

## 素材の選び方

見る試合は選ぶ。優先順位はこうだ。

- 負けた試合 > 格上相手の試合 > 惜敗した試合
- 勝った試合の粗探しは後回しでいい

リプレイのまま見るより、動画に書き出してから見るほうがいい。区間指定や見返しがやりやすくなる。見る速度は0.5倍速で2〜3回。1回目はざっと違和感を拾うだけでよく、2回目以降でその場面を深掘りする。

## 全部見ない。撃墜シーンを優先する

1試合を頭から全部見直すのは時間がかかる割に得るものが少ない。一番差が出るのは「死んだ場面」なので、そこだけ深掘りするのが効率がいい。

撃墜シーンは次の3ステップで深掘りする。

1. 何の技で死んだかを特定する
2. 撃墜の1〜3手前まで遡り、起点になった行動を探す
3. 原因をタグ分類する

タグは次の7つから選ぶ。

- 立ち回り被撃墜
- 着地時
- 復帰中
- 崖上がり時
- コンボ完走許容
- 暴発・自滅
- %管理ミス

このタグは、試合を「にらみ合い(neutral)」「有利」「不利」「崖の攻防」という4つの状態で捉える見方(→[状態遷移で考える立ち回り](state-transitions))にもだいたい対応する。「着地時」なら不利の状態、「崖上がり時」なら崖の攻防、という具合だ。

## 実行ミスと判断ミスを分ける

死因が分かったら、それが実行ミスか判断ミスかを分ける。ここを混同すると練習の方向を間違える。

- **実行ミス**: 意図した入力が出せなかった。頭では正しい選択ができていたのに、指が追いつかなかったケース。これはトレーニングモードの課題にして、振り返りでは深追いしない。
- **判断ミス**: 選択そのものが悪かったケース。次の4つを自分に問う。
  1. なぜその判断をしたか
  2. その時の心理状態はどうだったか
  3. 相手は何を考えていたと思うか
  4. 次はより良い選択肢があったか

## One-Mistake Rule

1回の振り返りで直すのは、最も頻繁でインパクトが大きいミス1つに絞る。多くても1〜3個まで。気づいた点を全部一気に直そうとすると、結局どれも身につかない。

やり方はシンプルだ。

1. 改善仮説を1文にまとめる（例:「崖の上がりで回避が多いので、その場上がりも混ぜる」）
2. 次にプレイするときは、その1点だけを意識する
3. できたかどうかは次の振り返りで確認する

## チェックリスト

状態別に、振り返りで問うべきことをまとめておく。

**neutral**

- 何がきっかけで不利に転落したか
- その行動は理由のある選択か、それとも手癖か
- リスクとリターンは見合っていたか

**有利**

- コンボ・確定反撃を取りこぼしていないか
- 着地狩り・崖狩りの継続を自分から切っていないか
- なぜ有利を手放したか

**不利**

- 復帰ルートが一直線で読まれていないか
- ジャンプを無駄遣いしていないか
- 崖を掴まず上から降りて狩られていないか
- 焦って大技を振っていないか

**崖攻防**

- 攻め側: 相手の上がり方の癖を読めたか、ライン管理はできていたか
- 守り側: 上がり方を散らせたか、同じ択の反復になっていないか

## smash-labのAIレビュー機能を使う

smash-labには、この振り返りをAIに手伝わせる機能がある。使い方は次の通り。

1. `/review` フォームでYouTube URLと、撃墜シーンの5〜10秒前の動画時刻を`mm:ss`形式で登録する（複数登録可）
2. Mac側のClaude Codeが該当区間を分析する
3. できあがった分析結果（findings=気づきのカード）を確認し、内容が的確なら承認、外していれば棄却する
4. 分析結果には「今回直す1点」が添えられる。これがOne-Mistake Ruleをそのまま実装したものだ

自分の目で見た違和感と、AIが拾った違和感を突き合わせると、見落としに気づきやすい。
